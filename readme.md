# 🏛️ Сайт банного комплекса «1000 и 1 ночь»

Современный, быстрый и отзывчивый веб-сайт для премиального банного комплекса **«1000 и 1 ночь»** (г. Екатеринбург). Проект спроектирован с упором на визуальное превосходство, плавные анимации и максимальную производительность.

---

## 🛠️ Стек технологий

*   **Фреймворк:** [Next.js 15.1.9](https://nextjs.org/) (с использованием **App Router**).
*   **Библиотека рендеринга:** [React 19.2.0](https://react.dev/).
*   **Типизация:** [TypeScript](https://www.typescriptlang.org/).
*   **Стилизация:** [Tailwind CSS v4](https://tailwindcss.com/) — используется новейшая версия с поддержкой импорта `@import "tailwindcss";` и директивой `@utility`.
*   **Анимации:** [Framer Motion](https://www.framer.com/motion/) — для премиальных переходов, зум-эффектов и адаптивного слайд-шоу.
*   **Тестирование:** [Playwright](https://playwright.dev/) — для автоматизированных e2e тестов.

## 📐 Архитектура проекта (Feature-Sliced Design)

Проект строго следует методологии **Feature-Sliced Design (FSD)**. Весь исходный код приложения расположен в директории `src/` и разделен на слои:

*   📂 `app/` — Инициализация приложения, глобальные стили (`styles/globals.css`), макеты (`layout.tsx`) и страницы (роуты).
    *   `rooms/[slug]/page.tsx` — динамический роут для отображения конкретного зала. Поскольку проект экспортируется статически, здесь используется `generateStaticParams` для пре-генерации путей на этапе сборки.
*   📂 `widgets/` — Крупные самостоятельные блоки интерфейса, собираемые из компонентов слоев `features` и `entities`.
    *   Примеры: `header`, `footer`, `room-display` (полноэкранный блок презентации зала).
*   📂 `features/` — Интерактивный функционал, приносящий пользу пользователю (действия с бизнес-логикой).
    *   Примеры: `room-slideshow` (галерея слайдов для комнат), `view-room`.
*   📂 `entities/` — Бизнес-сущности приложения, инкапсулирующие данные, типы и базовую логику.
    *   `room` — сущность банного зала (модели данных, список залов, хуки для выборки).
    *   `feature` — характеристики зала (бассейн, бильярд, тип парной, караоке и т. д.).
*   📂 `shared/` — Переиспользуемый инфраструктурный код, независимый от бизнес-логики.
    *   `ui/` — простейшие UI-компоненты (кнопки, иконки).
    *   `config/` — глобальные конфигурации приложения.
    *   `lib/` — вспомогательные чистые функции, хелперы.

## ⚙️ Важные особенности конфигурации

### 📦 Режим Static Export (Статический экспорт)
Проект настроен на сборку в полностью статичные HTML/CSS/JS файлы.
В файле [next.config.mjs](file:///Users/eq/Documents/sasauni/next.config.mjs) включена опция:
```javascript
output: 'export'
```
*   **Результат сборки:** Все файлы компилируются в автономную директорию `out/`.
*   **Оптимизация картинок:** Поскольку проект работает без Node.js сервера (в режиме статики), стандартная динамическая оптимизация Next.js изображений отключена (`images.unoptimized = true`). Все изображения отдаются как есть.
*   **Динамические роуты:** Для генерации статических HTML-страниц для каждого зала в `src/app/rooms/[slug]/page.tsx` обязательно реализован метод `generateStaticParams()`.

## 🚀 Запуск и Разработка

Перед началом работы убедитесь, что у вас установлен **Node.js LTS** (рекомендуется версия **20** или выше) и пакетный менеджер **npm**.

### 1. Установка зависимостей
```bash
npm install
```

### 2. Запуск локального сервера разработки
Запуск локального сервера с поддержкой Hot Module Replacement (HMR):
```bash
npm run dev
```
Сервер будет доступен по адресу: `http://localhost:3000`

### 3. Проверка типов и линтинг
Перед отправкой изменений в репозиторий рекомендуется запустить автоматическую проверку:
```bash
# Валидация TypeScript типов (без генерации файлов)
npm run type-check

# Проверка кода линтером (ESLint)
npm run lint

# Автоматическое исправление ошибок стиля кода
npm run lint:fix
```

## 📦 Сборка и Развертывание (Деплой)

Проект развертывается как полностью статический сайт на хостинге **Reg.ru**.

### 1. Генерация статической сборки
Запустите команду сборки:
```bash
npm run build
```
После успешного завершения процесса в корневой директории проекта будет создана папка `out/`. Она содержит оптимизированный статический HTML, CSS, JavaScript и медиафайлы.

### 2. Загрузка на хостинг Reg.ru
Для обновления сайта выполните следующие шаги:

1.  Подключитесь к вашему хостинг-аккаунту Reg.ru (через панель управления хостингом ISPmanager / cPanel / Plesk или по протоколам SFTP/FTP).
2.  Перейдите в корневой каталог вашего домена (обычно это папка `www/1001ночь.site` или `public_html`).
3.  Очистите старое содержимое этой папки (за исключением системных файлов хостинга, если они необходимы).
4.  Загрузите **всё содержимое папки `out/`** (но не саму папку `out`!) в этот корневой каталог.

> [!IMPORTANT]
> Переносить нужно именно содержимое папки `out/` (так, чтобы файлы `index.html`, `404.html` и папка `_next` оказались непосредственно в корневой директории вашего сайта на хостинге).
